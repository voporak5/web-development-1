IDE (Integrated Development Environment)

Visual Studio Code

Probably the easiest thing you'll find installed on the machines at school. Very common especially since it isn't exclusively for web development. Don't confuse this with the standard Visual Studio which you can use for web development, but Visual Studio Code is much more friendly.

Brackets

I highly recommend this one because the autofill is very smart and it has other cool features like showing you the color in a preview as you're typing the color out.

Notepad++

This gets the job done and lets you focus on typing out your HTML without distracting you from other windows, editor features, etc. because it's just a better version of Notepad intended for programming. Also it also has intellisense if you write PHP. Brackets does this too but you have to set it up.

TextEdit

Mac version of Notepad so definitely not ideal. It also tries to save your files as rich text files (.rtf) which is just a text file (.txt) with the ability to handle formatting similar to a word doc.

Notepad

Plain text editor on Windows that definitely will allow you to get the job done but you need to know what you are doing since there are no autofill features. Be sure you save your files as .html

Github


Github is what we call a repository service. You can think of a repository service similar to Dropbox, Google Drive, OneDrive, etc. except the primary intent is for storing code. You can actually store whatever files you want, but there are special functions developed around the assumption you're using it to store code. Github provides other features that this class is in no way meant to go over. An additional feature that Github provides is the ability to host a website that you've been developing. The functionality of the website can be pretty limited, but for this class it provides everything we care about. There are other services such as Bitbucket that provide the same or similar features, but Github is the most common.

Web Development Basics


Anatomy of an HTML file 

<!--This is a comment. Begin writing a comment with a lt sign, an ! and 2 hypens. End with 2 hypens and a gt sign -->
<!--Everything in between will not be processed-->
<!--
    You can also enter line returns
    and everything in between will still be ignored
-->
        
<!--First specify the type of document using <!doctype html> -->
<!--This helps the webpage process conistently across different web browsers-->
<!doctype html>
        
<!--Then specify the language to help search engines correctly find your page-->
<!--This also serves as the beginning of the main code you will write-->
<html lang="en">
    
    <!--Inside the html section we need to declare a section called head-->
    <!--The head section is where we declare information about the website as-->
    <!--well as load in external resources which we will look at in the future-->
    <head>
    
        <!--First declare how much data to allocate in the web browser (don't think about this too hard, just do it)-->
        <meta charset="utf-8">
        
        <!--Then declare how the content will scale-->
        <meta name="viewport" content="width=device-width, initial-scale=1">
        
        <!--Lastly name your page (this will appear on the tab of the web browser page. You can see this web page says Week 1)-->
        <title>Homepage</title>
        
    <!--Make sure to declare the end of the head section-->
    </head>

    <!--After the head section we then declare the body section-->
    <!--All of the code representing the content of the web page will go inside of here-->
    <body>
    
        <!--As an example we have the words Hello, world! inside of an h1 tag which makes it appear very large-->
        <h1>Hello, world!</h1>
        
    <!--This is where we say we are done writing content for the web page-->
    </body>

<!--This is where we say we are done writing HTML-->
</html>

Anatomy of an HTML tag


In order to begin building pages you have to get familiar with tags.
Tags are a way for you to define what you want to show on a web page.
You start with a < sign, the name of the tag, and a > sign. This is called the opening tag.

Then you enter your content.

Then you enter the a closing tag which is just like the opening tag but with a / like the below example. 
<h1>
    Hello World
</h1>
Not every tag requires a closing tag. The meta tag for instance just configure your
web page and does not apply any effect to content.

Speaking of the meta tag, you may notice it has extra stuff after the word "meta"
These are called attributes and there are many different kinds that don't all
necessarily work for every single tag so you'll need to do the research to figure out
what is required/possible 
<meta name="viewport" content="width=device-width, initial-scale=1">
