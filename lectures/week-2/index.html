Anatomy of URL

  • Uniform Resource Locator

    • https://www.google.com downloads a resource that lives on Google's servers which you can use to search for web pages on the internet

  • Broken up it consists of

    • Also referred to as the protocol
    • http
    • https
    • sms
    • email
    • tel
    • Any of the above keywords followed by ://
    • There are ones that can even open apps on your Android or IOS device
    • www
    • The part that immediately follows the Scheme
    • www is the standard default but you can easily create your own through your web hosting service
    • Maybe your website offers many services and features so you could do shop.mywebsite.com and profile.mywebsite.com to help organize these services and features
    Website extension such as .com .org .gov etc.
    Actual website name such as google facebook github etc.
    The path that comes after the extension characterized by folders www.google.com/folder-1/folder-1-1/folder-1-1-1
    Series of key/value pairs at the end of the url that starts with ?. Useful if you have a website that is going to behave differently based on the data passed in such as if you passed in a user id maybe the web page will show that user's account.
    The file being loaded by the url such as index.html profile.php homepage.js etc.
    The data for that query that follows the ? characterized by a series of key-value pairs such as key1=value1&key2=value2 or facebook.com/account.html?email=ccintron@colum.edu&password=rickroll
    At the very end you can put a # and some value to jump to a specific page. Click Me to try.
    Goes after the extension and is characterized by a colon and some number such as facebook.com:443

    Different ports have different meanings and purposes. 80 is for http and 443 is for https

Formatting Code


It's important to make your HTML readable and organized not only to make it easier for any team members to understand what your code is doing, but also to help you see your website from the code side.

Here is an example of nicely formatted code 


<html lang="en">
    <head>
        <title>Home</title>
    </head>
    <body>
        <div class="nav">
            <a href="index.html>Home</a>
            <a href="index.html>About Me</a>
            <a href="index.html>Resume</a>
        </div>
        
        <div class="content">
            <h1>My Homepage</> 
            <br>
            <div class="portrait">
                <img src="imgs/art-1.png">
                <p>Here is my art</p>
            </div>
        </div>

Links

Creating a Link is really easy.
We use the "a" tag and we have to at least specify the href attribute to define a destination. 

<a href="https://www.youtube.com/watch?v=OkXjowZ_JgA" target="_blank">Click Me</a>

You'll also notice the "target" attribute
This specifies where the web page should open (this tab, a new tab, a new window, etc.)
  • _blank
    • Opens in a new window or tab
  • _self
    • Opens in the same frame (more on this in a later lecture but for now it's the same page)
    • The default value if you don't specify the target attribute
  • _parent
    • Opens in parent frame (more on this in a later lecture)
  • _top
    • Opens in the full body of the window
  • framename
    • Opens in a specified frame (more on this in a later lecture)

When specifying a destination we can either specify an absolute url or a relative url.

An absolute url is the full url destination such as the one shown in the web browser https://voporak5.github.io/web-development-1/lectures/week-2/index.html

A relative url is based on where files are stored on your website. Rather than having to type out the full absolute url to open a web page or other files, you just have to specify the path to get to that resource based on where the current web page is stored

Absolute Week 1 

<a href="https://www.youtube.com/watch?v=OkXjowZ_JgA" target="_blank">Click Me</a>

Relative Week 1 

<a href="../week-1/index.html" target="_blank">Click Me</a>

Both links work but look a little different. If you move your entire website to a new location then you won't have to go and fix all of your links.

Additionally, files will load faster since the link doesn't have to redirect around the world to get back to the server the website is stored on.

Breaking down the relative url...
  • directory (folder) changes are separated by /
  • to exit a directory you put ..
  • to enter a directory you put the name of the directory
  • to open the webpage, image, file, etc. put the name of it and the extension at the very end (home.html, styles.css, portrait.png, etc.)
  • brackets will show what folders you can open as you're typing which is cool
  • open the file explorer (windows) or finder (mac) and look at your file structure and folders to see what your file path needs to be
  • if the filepath shows something like c:// at the beginning you're going to lose points because that is an absolute url based on your computer and that will not work when someone (me) clicks your link on GitHub
  • spaces in the url are big no no's
    • do not do it
      • you will not be happy
        • do not do it
          • put hypens instead of spaces in folder and file names

You can also create links that jump to different parts of your web page. If you want to jump to the top of the web page simply put # in the href value.

Try clicking "Return To Top" 

<a href="#" target="_blank">Return To Top</a>

You can add the id attribute to any element and specify a name. Then in front of the # of your link you add the name of that id you specified for the element you want to jump to. Click Me to jump to Formatting Code.

You need the put # in the link to tell the web page to go to this location on the current web page and don't try to open a new page. 

<h2 id="formatting-code">Formatting Code</h2>
    
<a href="#formatting-code" target="_blank">Click Me</a>

Images

Loading an image is easy. To do so we use the <img> tag.

The <img> has several attributes to configure the image 

<img src="https://placehold.co/128x128" width=128px height=128px alt="128x128 image">

<img src="https://placehold.co/128x128" width=128px height=128px alt="128x128 image">

  • src
    • Location of the image that is being loaded
    • Can be relative or absolute (the above images are absolute and being loaded from another website)
      • They do take longer to load though and it would be more efficient to store them in the website files and load them with a relative path
  • height
    • How tall the image should be
    • If you don't set this then the image will be whatever the actual image height is or scaled based on what you set for the width
  • width
    • How wide the image should be
    • If you don't set this then the image will be whatever the actual image width is or scaled based on what you set for the height
  • alt
    • Text that will show in the image area if the image fails to load (see the above example that says "Missing image")
Be careful when setting width and height. What looks good on a smaller window might look terrible on a larger one.


We will look at how to handle this problem next week. If you change the size of the web browser you will see the above images correctly scaling up/down.

You can also add captions easily to your images by wrapping the image with the <figure> tag. To specify the caption you can wrap your text with the <figcaption> tag. This needs to be inside of the <figure> tag contents (See the below reference code). 

<figure>
    <img src="https://placehold.co/256x256">
    <figcaption>A placeholder image I grabbed</figcaption>
</figure>

There's 1 problem with loading images and that is they can take a long time to load. However, we can pack them into a single image and load them all at once. We'll look at this next week but you can read more about it Here if you are interested.


Lists


There are 3 types of lists we can create: ordered, unordered, and definition lists. For ordered lists and unordered lists you can use the <ol> and <ul> tags respectively.

For any list item you want to add to either of those lists you can use the <li> tag.

  1. This
  2. Is
  3. An
  4. Ordered
  5. List
    1. You can try putting lists inside of lists but it doesn't make a 5.1 correctly without some custom stuff...Try googling it!


<ol>
    <li>This</li>
    <li>Is</li>
    <li>An</li>
    <li>Ordered</li>
    <li>List</li>
    <ol>
        <li>You can try putting lists inside of lists but it doesn't make a 5.1 correctly without some custom stuff...Try googling it!</li>
    </ol>
</ol>

  • List

</ul>

Definition lists are intended to be used like dictionaries where you specify a word with a definition that is cleanly formatted. This requires wrapping your list with the <dl> tag (definition list) then for each term you wrap it with the <dt> tag and lastly for each definition...definition you wrap it with the <dd> tag

Lionel Richie
Hello, is it me you're looking for
Billie Eilish
I'm the Bad Guy...Duh!
Smash Mouth
Somebody once told the world is gonna roll me
<dl>
    <dt>Lionel Richie</dt>
    <dd>Hello, is it me you're looking for</dd>
    <dt>Billie Eilish</dt>
    <dd>I'm the Bad Guy...Duh!</dd>
    <dt>Smash Mouth</dt>
    <dd>Somebody once told the world is gonna roll me</dd>
</dl>
