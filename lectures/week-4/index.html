Color

Up until now we have only talked about pre-defined colors (aqua, smoke white, white, earth green, etc.) however, there are many many more (16 million+ that computer can show).

Here are common color setting methods.

Red, Green, Blue (255, 128, 36)
Sometimes RGBA where the A is alpha and looks like this (255, 128, 36, 255)

If you want to change the alpha then you need to specify the opacity which is a value that ranges from 0-1 where 0 is transparent and 1 is completely opaque. You ocan also do a percentage from 0% to 100%. 

p {
    color: rgb(255, 128, 36);
    opacity: 0.5;
}

Cyan Magenta Yellow Key
It's for printing, not computers. If you see this option in illustrator when you make an image, don't select it.

It's still RGB/RGBA however it looks like this #ff8024ff which is the base 16 equivalent of (255, 128, 36, 255).

Programmers might want to type 0xff8024. Websites do not like this. You need to use the # symbol.

If you want to change the alpha then you need to specify the opacity which is a value that ranges from 0-1 where 0 is transparent and 1 is completely opaque. You ocan also do a percentage from 0% to 100%. 

p {
    color: #ff8024;
    opacity: 50%;
}

Hue, Saturation, Lightness, Alpha.

I have no idea how to figure out what color to make based on this method. You'd need to do research if you decided to use it but the most common methods are hex or pre-defined colors.

A benefit of this method is that you do not need the opacity property to set transparency. The 4th value represents alpha. 

p {
    color: hsl(0, 100%, 50%, 100%);
}


Changing Fonts

Changing fonts is easy with the font-family property 

<h4 style="font-family: cursive">
    Changing fonts is easy with the font-family property
</h4>

You can specify the font value in quotes or not 

<h4 style="font-family: 'Times'">
    You can specify the font value in quotes or not
</h4>

And you can specify many different fonts in the event of one of the fonts failing to load 

<h4 style="font-family: Verdana, Times, 'Courier New'">
    And you can specify many different fonts in the event of one of the fonts failing to load
</h4>

You don't need quotes however, it is recommended if the font name has special characters such as $ , ! or /

It's also good practice to end the list of fallback fonts with something generic (pages 269-270 of HTML & CSS Jon Duckett)

Why do you need fallback fonts though? Won't they all work?

Not if... A. the font isn't installed by the user, but also because of B. not every font works on every web browser!



Loading External CSS (From CSS file)


We use the link tag and add the rel attribute to specify what we're trying to load.

We also specify the href attribute to specify where the thing we're trying to load is located.

As before, you can specify an absolute link or a relative link to load a CSS file that's stored on your website or somewhere else.

You should store your css files in a folder called styles. For simplicity name your file styles.css as well. 


<head>
    <meta charset="UTF-8">
    <title>My Website</title>
    
    <link rel="stylesheet" href="styles/styles.css">
</head>

<body>
    <p>
        Hi My name is, What My Name is, Who My Name is, Chicka chicka Slim Shady.
    </p>
</body>

Loading Internal CSS (Without CSS file)


As an option you can write your CSS directly in your HTML file however, the CSS will only apply to that specific web page.

In order to do this you place the same CSS code inside of a <style> tag in the <head> 


<head>
    <meta charset="UTF-8">
    <title>My Website</title>
    
    <style>
        p {
            color: white;
            font-size: 32px;
        }
    </style>
</head>

<body>
    <p>
        Hi My name is, What My Name is, Who My Name is, Chicka chicka Slim Shady.
    </p>
</body>

Important Style Properties

Just like images, you can change the space that your contents covers by specifying width and height in your CSS.

The above div has it implemented as in inline style, but if you did it as external CSS then it would look like this. 

div {
    width: 25%;
}

In terms of values, you can specify exact units such as 25px or percentages such as 50%. If you specify a percentage then that would be 50% of the container's width. So, if we don't have any <div> tags on our web page and we set it to a width of 50%, then that would be 50% of our web page's width. If we created a <div> inside of that <div> and also set it to 50%, then that would be 50% of the parent <div> (which doing the math 50% of 50% is 25% of the screen width)

The next important style property for positioning content is margin.
Margins are the spacing between elements. You can specify margins on all 4 sides of an element.
The below example sets the space between the left and right edge of the parent containers to be 25% of the width respectively making the total container width 50% wide and because of the spacing on the left and right it appears centered.

Here is some text 

<p style="margin-left: 25%;margin-right: 25%; background-color: orangered;">
    Here is some text
</p>

Look what happens when we only set the left margin.

Here is some text 

<p style="margin-left: 25%; background-color: orangered;">
    Here is some text
</p>

Look what happens when we only set the right margin.

Here is some text 

<p style="margin-right: 25%; background-color: orangered;">
    Here is some text
</p>

Of course, the actual text is not centered inside of the container so we can specify the text-align property to fix that.

Here is some text 

<p style="text-align: center; background-color: orangered;">
    Here is some text
</p>

Maybe we want it centered and we want the positioning of the text to only be 50.

Here is some text 

<p style="text-align: center; margin-left: 25%;margin-right: 25%; background-color: orangered;">
    Here is some text
</p>

Don't be mislead by the name of the property text-align. It isn't intended to be exclusively used with text. We can use it on anything such as images. However, we need to wrap the image in a div and apply the property to the div. text-align only works if the element you are applying the style to is wide enough. Applying it directly to an image won't work because the element width and height is exactly what is specified. The <p> tag takes up the full width of the container that it exists in as a default. 

<div style="text-align: center">
    <img src="img/gow.jpg">
</div>

Without the div, we would need to use margins and play with the width. We don't know what percentage of width the image is, but we can force it to be a percentage and just calculate the difference in margins. So if the image is 30% of the width of the screen we can make the margins left/right be 35% each. (30 + 35 + 35 = 100 ) 

<img src="img/gow.jpg" style="margin-left: 35%; margin-right: 35%" width="30%">

You can specify the margins of each side all at once in your style or you can specify them individually. It is up to you. 

img {
    margin-top: 10px;
    margin-right: 25%;
    margin-bottom: 10px;
    margin-left: 25%;
}

If you specify it all at once then the order is top, right, bottom, left. 

img {
    margin: 10px 25% 10px 25%;
}

What about putting multiple elements in the same row?
Since the size of the image conforms to the actual image then images do this naturally.
