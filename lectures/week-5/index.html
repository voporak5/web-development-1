Fixed, Absolute, and Relative Positioning

Up to this point all of our elements have just been drawn from the beginning to the end and in that order. However, we can change where elements are drawn even if the code is written in a different place.

This is done by changing the position property in css.
Have you noticed that the nav at the very top of this web page stays fixed to the top of the browser window even when we scroll down?

This is accomplished by setting the value of position to fixed. It is possible that other elements will still be shown in front of it so we also need to change the z-index property which is resposible for layering elements.

Fixed conted is positioned relative to the web browser window. 

.fixed {
    position: fixed;
    z-index: 9999;
}

If we wanted our content to be positioned relative to the full web page (which may extend beyond the edge of the web browser window) and as opposed to where it's supposed to be drawn based on where we wrote HTML... (look in the top left corner)

then we set the position to absolute 

.absolute {
    position: absolute;
    top: 70px;
    left: 0px;
}

You'll notice the above style has a new property called top and left. You can add these to offset the element. I added them because the example was not correctly being positioned so I had to set the value of top to be 70px from the top and left 0px from the left.

Lastly, if we just want to slightly offset content from where it would be normally drawn, or rather relative to the position it's supposed to be drawn then we set the position to relative.

An issue you can see is that I've injected some CSS just on the word relative, however the . has not shifted as well. When you move elements with left and top, the white space does not become usable and is treated as if the element was not offset. 

.relative {
    position: relative;
    left: -50px;
}

Color

Up until now we have only talked about pre-defined colors (aqua, smoke white, white, earth green, etc.) however, there are many many more (16 million+ that computer can show).

Here are common color setting methods.

Red, Green, Blue (255, 128, 36)
Sometimes RGBA where the A is alpha and looks like this (255, 128, 36, 255)

If you want to change the alpha then you need to specify the opacity which is a value that ranges from 0-1 where 0 is transparent and 1 is completely opaque. You ocan also do a percentage from 0% to 100%. 

p {
    color: rgb(255, 128, 36);
    opacity: 0.5;
}

Cyan Magenta Yellow Key
It's for printing, not computers. If you see this option in illustrator when you make an image, don't select it.

It's still RGB/RGBA however it looks like this #ff8024ff which is the base 16 equivalent of (255, 128, 36, 255).

Programmers might want to type 0xff8024. Websites do not like this. You need to use the # symbol.

If you want to change the alpha then you need to specify the opacity which is a value that ranges from 0-1 where 0 is transparent and 1 is completely opaque. You ocan also do a percentage from 0% to 100%. 

p {
    color: #ff8024;
    opacity: 50%;
}

Hue, Saturation, Lightness, Alpha.

I have no idea how to figure out what color to make based on this method. You'd need to do research if you decided to use it but the most common methods are hex or pre-defined colors.

A benefit of this method is that you do not need the opacity property to set transparency. The 4th value represents alpha. 

p {
    color: hsl(0, 100%, 50%, 100%);
}


Changing Fonts

Changing fonts is easy with the font-family property 

<h4 style="font-family: cursive">
    Changing fonts is easy with the font-family property
</h4>

You can specify the font value in quotes or not 

<h4 style="font-family: 'Times'">
    You can specify the font value in quotes or not
</h4>

And you can specify many different fonts in the event of one of the fonts failing to load 

<h4 style="font-family: Verdana, Times, 'Courier New'">
    And you can specify many different fonts in the event of one of the fonts failing to load
</h4>

You don't need quotes however, it is recommended if the font name has special characters such as $ , ! or /

It's also good practice to end the list of fallback fonts with something generic (pages 269-270 of HTML & CSS Jon Duckett)

Why do you need fallback fonts though? Won't they all work?

Not if... A. the font isn't installed by the user, but also because of B. not every font works on every web browser!



Custom Fonts

In order to add a custom font we need to do 2 things.

First add it to our files. Create a folder called fonts in your website directory and place the font files.



Second you need to load the font with CSS. 

@font-face {
    font-family: 'Gazpacho';
    src: url('../fonts/Gazpacho-Black.woff2') format('woff2'),
         url('../fonts/Gazpacho-Black.woff') format('woff'),
    font-weight: 900;
    font-style: normal;
    font-display: swap;
}

You would place the above code directly in your CSS file. The 'font-family' property is the identifer you will declare for the font. If I want to apply the font to text then I would type Gazpacho. The reason there are multiple url values is because they are fallbacks if one ofo them does not work for a particular web browser. Additionally, the URL does not actually have to be pointing to a file in your directories. It can be a URL to a font that exists on another website. You might do this with Google fonts.

The below code is an example of an h1 tag using the loading font from the above code. 


h1 {
    font-family: Gazpacho
}

The easies way to generate the CSS associated with a font is to use a typeface generator. The book has a link to one on fontsquirrel but I checked and it doesn't work. As of now this one works Transfonter. You can also just Google "Font face generator css" to find more.

Interactive Styles


You have have noticed that <a> tags looks different depending on if you click them, put your cursor over them, or scroll past them. You can configure them by appending a : to the tag in the CSS and specifying either link,visited,hover, or active. Look at this link here 


a:link {
    color: #f95738;
}

a:hover {
    color: #adb5bd;
}

a:visited {
    color: #f95738;
}

a:active {
    color: #f95738;
}

active is the state when you've actively clicked the link.

Some of these states are not exclusive to links.

Creating Space In Elements


So far we've only talked about creating space between elements using margins.
We can actually create space inside elements to push our content around


Without padding

<div style="background-color: aqua; width:200px;height: 200px;>
    <div style="background-color: brown; width:50px;height: 50px;></div>
</div>

With 75px padding on the top and left 

<div style="background-color: aqua; width:200px;height: 200px; padding-left: 75px; padding-top:75px;>
    <div style="background-color: brown; width:50px;height: 50px;></div>
</div>


The padding should be applied to the container element because it pushes the content on the inside around.

You can combine as many different classes as you want. Be sure to separate each class in your class attribute with a space. If the class itself has spaces then consider using a hypen - to act as the space such as instead of "h4.green text" make it "h4.green-text".

Another thing to note is the above method for creating classes is specific to each tag. If you tried to apply "red left" to an <h3> then it will not work because the above definition is specific to the <h4> tag. You will have to make a class definition for that tag in your css if you want it to work. You can comfortably use the same names as well without running into a conflict.
