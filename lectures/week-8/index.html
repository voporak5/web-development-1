Loading External Javascript

Last week we looked at injecting Javascript using the <script> tag. On a small webpage this is fine, but on a larger page or website that is more complex this could be tedious, messy, unorganized, and difficult to read much like it is with styles that are not placed in a CSS file.

We can work around this in the same way we do with our CSS by putting our Javascript in an external file.

Create a new file that ends with the .js extension and place it in a folder called scripts. You can download this example.js file.

In order to load the file you must add similar code to how we load CSS except the key difference is we place this code in the <body> 



<body>
    <script src="scripts/example.js"></script>
</body>

Something else to note is that you should place the code to load external js files at the very end of the body. This way the web page can load as fast as possible. Otherwise, the web page will stop and wait until the script is loaded before moving on. Additionally, this will guarantee that all elements are loaded which is important if your javascript relies on something such as a button.

Do This 

<body>
    <h1>Hello</h1>
    <script src="scripts/example.js"></script>
</body>

Not This 

<body>
    <h1>Hello</h1>
    <script src="scripts/example.js"></script>
</body>

Getting User Input

We can collect information from the user by using tags meant for creating a form and pass it between pages as the query part of the url (or POST data which is way advanced for this class).

Here are the types of inputs we have access to
  • Text Input (Text Box)
  • Password Input (Text Box with hidden character)
  • Text Areas (Big Text Box)
  • Buttons
  • Radio Dials (Multiple choice selection limited to 1 selected item)
  • Checkboxes (Multiple choice selected allowing multiple selected items)
  • Dropdown List
  • Submit Button (Collects data and goes to next page)
  • Image Button (Same as submit but is an image)
  • File Upload Button (Destination page needs extra code to handle the file)






If you want to create a collection of elements for collecting data that will be passed onto another page for processing then you can start by declaring the <form> tag and add the action attribute to specify the page that should be loaded when the submit button is pressed.

Just like with <a> tags you can add the target attribute to open a new web page instead of using the current page. 

<form action="submit.html" target="_blank">
  
</form>

Inside the <form> you start adding all of the input types you wish to use 

<form action="submit.html" target="_blank">
    <input>
    <input type="date">
    <input type="submit">
</form>

Although that is all you need to create input fields, if you want to pass the data to another web page then you will need to add the name attribute to each input field which will allow the input values to be appended to the end of the URL as a part of the query.

Since this data is being added to the URL we should avoid spaces in the name. Any spaces in the actual input will be replaced with + 

<form action="submit.html" target="_blank">
    <input name="firstName">
    <input type="date" name="date">
    <input type="submit">
</form>

Assuming the values for firstName was Cameron and the date was 2024-05-30 then the URL that would be loaded would look like submit.html?firstName=Cameron& date=2024-05-30

You can look at the form at the beginning of this web page for a concrete demonstration.

Some input types require a second attribute called value because they don't have a way for the user to specify a value or because they are pre-defined such as in a dropdown list.

Consider a situation where you want something different to happen depending on which of 2 buttons are pressed for an account edit page. Maybe 1 button saves any changes and the other button deletes the account. 

<form action="submit.html" target="_blank">
    <input type="submit" value="save">
    <input type="submit" value="delete">
</form>
